E-INK CALENDAR

The e-Ink calendar project stemmed from a need to have a low power calendar device linked to the internet that was battery operated and therefore could be put anywhere. At work we were having difficulty managing people accessing the PCs when others were remotely logged in. I thought a simple calendar showing the booked timeslots would help prevent people from logging in when others had booked the setup and were remotely using it (because you cannot tell from looking at PC if someone is remotely connected to it already). The image below shows the project concept.

  • Waveshare 2.7" display
  • Rasperry-Pi 3
  • Wifi Connection to access Google Calendar

E-Ink Working Principle

E-Ink is the perfect solution for a low power display that does not need to update very often. It works by having many little cells with differently charged black and white particles. The display manipulates the color by applying a voltage to the top or bottom electrode which moves the charged particles. Because the display has phyisical moving particles it's relativley slow compared to LED/LCD displays and therefore wouldn't be a good candidate for a TV or computer monitor, however, it has found its niche in ebooks.

Drawing Image

The e-Ink display basically shows a picture, so in order to have it depict a calendar we have to create the calendar image programatically. I used python library PIL to generate the image section by section (you can view the code here). When you finished creating the image object you send it over SPI communication to the e-Ink display and tell it to update. The rest is handled by the display itself with the library given by the manufacturer.

Runtime Software Flow

I wanted to link the calendar to a service that is widley used and since google has an easy to use API that is what I preferred. The user would log into their google account and save their credentials (which are encrypted) on the device so that when the device trys to ping the google server to retrieve the data it has the proper authorization. After that, the device would ping google once every 5 minutes to check to see if there had been any updates to calendar events, if there were changes it would update the display with the new info.
