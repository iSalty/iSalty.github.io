Stop Sign Detector Program

I wanted to learn more about computer vision and get a deeper understanding of the difference between rule based programs and AI programs that use deep neural networks to learn and classify images. I came up with the idea of writing a program that can detect stop signs in a given image using a rule based approach first and then again making the same program using machine learning and neural networks. The program shown here is the rule based approach which proved to be more effective than I had anticipated. Shown below is an example input image.

Find the red pixels

After you load the image we apply a mask to find all of the red pixels (since all stop signs are red in the US). We convert the image data to HSV (Hue, Saturation, Value) for easier color interpretation and create masks which get us only the red pixels. Below shows the input image with the red mask applied, you can see it gets the stop sign and the brake lights. We can then apply pattern matching on this more simplified image.

Find Contours

Now that we have a more simplified image to deal with, we can find contours on this image which picks out lines and shapes and groups them. To do this we use some built in functions from the OpenCv library. The picture below shows the contour lines in green, and it makes it easy to see that it is grouping the edges on the black/red boundary.

Determine if Stop Sign

To make a decision on whether there is a stop sign in the image I just used 2 simple rules: Is it Red? Is it an Octagon (8 sides)? If both of these conditions are met then it determines that it is a stop sign. I have a function in the code which can determine the number of sides that a given contour has, it loops through all of the contours in the image and finds if there is one with 8 sides. If yes, it draws a box around that contour and shows it on the original image saying it found a stop sign. This same method would work on a video, just apply to each frame.
