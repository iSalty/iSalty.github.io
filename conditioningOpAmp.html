Signal Conditioning Op Amp

At Ford we were testing some lighting features using the ambient light nodes (RGB LEDs around the cabin) and wanted to automate the testing. In order to do so, we needed to tap into the PWM signals so we could read each duty cycle to determine the color and intensity. However, when we plugged in the signals to our PWM board we ran into several problems. I was able to fix these problems using an Op Amp ciruict between the LED and our I/O board.

Stage 1: Buffer

When we first connected the LED's PWM pin directly to our I/O board we noticed that the LED was dimly lit, this mean that there was some parasitic current passing through the I/O board. We found out that the PWM board only had an input impedance of 300k Omhs. Therefore, the first stage of my amplyifier would have to be a buffer which, ideally, has infinite input impedance and would fix this problem.

Stage 2: Low Pass Filter

Another issue was the fact that the PWM signal was biased up by the battery voltage (12V) and the forward voltage of the red LED was only 1.7V. This made it very difficult to set a threshold voltage for the PWM card to trigger from. I wanted my circuit to remove the VBatt bias so I would have a PWM signal roughly between 0V and 5V. I used a voltage divider and Low Pass filter in between the two Op Amps to remove some bias and get rid of any unwanted high frequency noise.

Stage 3: Non-Inverting Amplifier

The third and final stage was the non-inverting amplifier with a gain of 3.2V/V. I wanted to amplify the peak to peak voltage to make it easier for the PWM card to read the signal properly because it is less likely that noise will put the signal over/under the threshold voltage. This was especially important for the Red LED because it had the lowest peak to peak voltage (1.7V) and was therefore the toughest to get a good reading of.
